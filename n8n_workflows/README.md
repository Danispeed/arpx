# ARPX Orchestration Engine

This folder contains the n8n workflow export (`arpx-mvp.json`).

## Setup

### 1. Start the application

From the project root:

```bash
docker compose up -d
```

n8n is available at `http://localhost:5678`.

### 2. Import the workflow

In n8n: **Workflows** → **Import from File** → select [`arpx-mvp.json`](arpx-mvp.json).

### 3. Add Azure OpenAI credentials

Create one credential:
- **Credentials** → **Add Credential** → **Header Auth**
- Header name: `api-key`
- Header value: Azure OpenAI API key

Assign the credential to each of the following nodes:
- `PlannerAgent`
- `ExplainerAgent`
- `MermaidAgent`
- `QuizAgent`
- `ImagePromptAgent`
- `ChatAgent`

### 4. Publish the workflow

Click **Publish** in the workflow editor.

## Prompts

[`prompts.yaml`](prompts.yaml) is the runtime source of truth for all agent prompts. The workflow reads `/data/prompts.yaml` on every execution — prompt changes take effect immediately without re-importing the workflow JSON.

Agents and their prompt keys:
- `PlannerAgent` — `planner.system`, `planner.user_template`
- `ExplainerAgent` — `explainer.levels[N].system` (per-level system prompt), `explainer.user_template`
- `MermaidAgent` — `mermaid.system` (level-aware complexity rules embedded), `mermaid.user_template`
- `QuizAgent` — `quiz.system` (level-aware question rules embedded), `quiz.user_template`
- `ImagePromptAgent` — `image_prompt.system` (level-calibrated style rules), `image_prompt.user_template`
- `ChatAgent` — `chat.levels[N].system` (per-level system prompt), `chat.user_template`

## API Contract

- **Method:** `POST`
- **URL:** `http://localhost:5678/webhook/arpx/orchestrate`
- **Header:** `Content-Type: application/json`

### Stage: `explain`

Request body:

```json
{
  "paper_excerpt": "Text chunks from Weaviate",
  "level": 3,
  "topics": ["Topic A", "Topic B"],
  "stage": "explain"
}
```

Fields:
- `paper_excerpt` — source text chunks retrieved from Weaviate.
- `level` — integer 1–10; selects the level-specific system prompt.
- `topics` — topic labels used as additional context.
- `stage` — `"explain"` (default when omitted).

Response body:

```json
{
  "text_explanation": "Adaptive summary text...",
  "mermaid_code": "flowchart TD\nA[Concept] --> B[Detail]",
  "quiz": "{\"questions\":[{\"question\":\"...\",\"options\":[\"A\",\"B\",\"C\",\"D\"],\"answer_index\":0,\"rationale\":\"...\"}]}",
  "image_prompt": "A clean isometric illustration of interlocking gears...",
  "analogy_image": "<base64 encoded PNG>",
  "planner_brief": "Core idea: ... Metaphor: ... Focus: ... Key points: ...",
  "debug": {
    "stage": "explain",
    "route": "orchestrate"
  }
}
```

Fields:
- `quiz` — JSON string containing a comprehension quiz with level-calibrated questions (3 for L1-3, 4 for L4-6, 5 for L7-10).
- `image_prompt` — image generation prompt generated by `ImagePromptAgent`. Empty string if the image service is unavailable.
- `analogy_image` — base64-encoded PNG from the GPU service. Empty string if unavailable.
- `planner_brief` — coordination brief from `PlannerAgent`, shared across `ExplainerAgent`, `MermaidAgent`, `QuizAgent`, and `ImagePromptAgent` for thematic cohesion.

Execution order: `PlannerAgent` → (`ExplainerAgent` + `MermaidAgent` + `QuizAgent` + `ImagePromptAgent` → `CallClusterAPI`) in parallel → merge.

### Stage: `chat`

Request body:

```json
{
  "stage": "chat",
  "paper_excerpt": "Text chunks from Weaviate",
  "level": 3,
  "query": "What does the attention mechanism actually do here?",
  "history": [
    { "role": "user", "content": "Can you explain the model architecture?" },
    { "role": "assistant", "content": "The model uses a transformer-based..." }
  ]
}
```

Fields:
- `stage` — must be `"chat"`.
- `paper_excerpt` — freshly retrieved chunks based on the current query (not cached from the explain stage).
- `level` — selects the level-specific chat system prompt.
- `query` — current user question.
- `history` — all prior turns as `{ "role": "user"|"assistant", "content": "..." }`. Send `[]` for the first message.

Message list sent to the model:

```
[system prompt (level-specific)]
...history (previous turns, oldest first)
[user: rendered chat template with paper_excerpt, level, and query]
```

Response body:

```json
{
  "text_explanation": "The attention mechanism computes...",
  "mermaid_code": "",
  "debug": {
    "stage": "chat",
    "route": "orchestrate"
  }
}
```

`mermaid_code` is always an empty string for chat responses. The caller appends both the user query and the assistant response to its local history array before the next request.

### Stage: `ping`

```bash
curl -sS -X POST "http://localhost:5678/webhook/arpx/orchestrate" \
  -H "Content-Type: application/json" \
  -d '{"stage":"ping"}'
```

Response: `text_explanation: "pong"`, `mermaid_code` with a test diagram, `debug.route: "ping"`.

Used by `supervisor.py` as a health check before every explain call.
